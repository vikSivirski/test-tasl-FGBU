# Test Task: Posts App

Небольшое SPA-приложение на `React + TypeScript` с использованием `TanStack Query` и `React Router`.

## Что реализовано

- Получение постов с `https://jsonplaceholder.typicode.com/posts` с параметрами `_limit` и `_page`.
- Вывод по 10 постов на страницу.
- Пагинация списка постов.
- Страница отдельного поста.
- Корректный возврат к ранее выбранной странице списка (`?page=` в URL).
- Состояния загрузки, ошибки и empty-state.
- Базовые unit-тесты для утилит пагинации и парсинга query-параметров.

## Архитектурный подход

Использована структура с разделением на `pages / widgets / entities / shared`:
- `entities` — доменные сущности и API;
- `widgets` — составные UI-блоки (список, пагинация);
- `pages` — страничные сценарии;
- `shared` — общие утилиты, конфиги и переиспользуемые компоненты.

## Комментарий по UI-библиотекам

В реальном коммерческом проекте для пагинации я, как правило, использовал бы готовые компоненты из UI-библиотеки (например, `Ant Design`, `MUI` и т.д.) для ускорения разработки и единообразия интерфейсов.

В рамках этого тестового задания UI-библиотеки не требовались, поэтому пагинация и связанная логика реализованы вручную — чтобы продемонстрировать понимание кода, архитектуры и работы с состояниями.
